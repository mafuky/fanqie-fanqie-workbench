# 章节区明暖色调视觉优化设计文档

## 概述

本轮优化只针对书籍页中的章节区做**纯视觉美化**，不改交互逻辑、不改接口、不改处理流程，也不扩散到全站。目标是把当前偏“后台列表”的章节区，调整为更像**稿纸 / 章节便签 / 作者桌面卡片**的写作产品视觉。

## 约束

- 只做界面美化，不修改功能逻辑
- 不影响另一个 agent 正在进行的工作范围
- 仅修改 `src/web/pages/books-page.tsx` 中章节区相关视觉表现
- 保留现有交互：展开/折叠、阶段筛选、处理按钮、处理中状态、日志区
- 保留现有品牌名“小番茄写作”，但本轮不强化番茄果味感

## 视觉方向

整体采用 **“纸张米杏”明暖风格**：
- 主基调从冷灰黑转为暖白、米杏、浅褐
- 章节块像稿纸上的章节便签，而不是管理面板中的表格行
- 暖色以柔和为主，不使用大面积高饱和橙红
- 趣味来自写作氛围，而非装饰性图形

关键词：
- 稿纸感
- 章节卡片感
- 作者桌面感
- 温润、轻巧、可写作

## 本轮只做的 4 个视觉改动

### 1. 章节行改成暖纸感卡片

当前章节区每一行更像扁平列表项。本轮将每章调整为“轻卡片”视觉：
- 背景色从冷色背景改为米杏/暖白层次
- 增加更柔和的圆角
- 使用轻微暖色阴影或内外边界，模拟纸页层级
- 卡片之间拉开一点呼吸感，减少纯表格感

**目的：** 让用户感觉自己在管理“章节稿件”，而不是在操作后台条目。

### 2. 章号改成“小标签”

当前章号只是普通数字显示。本轮将章号改成更有趣的小标签，例如：
- `第12章`
- 暖色胶囊形标签
- 颜色比正文更浅，像贴在卡片左侧的小便签

**目的：** 强化章节感，让每一章更像独立创作单元。

### 3. 状态 Badge 全部暖调化

当前阶段 badge 中有偏冷的蓝色/紫色，本轮统一收敛到更暖、更柔和的系统：
- 待写作：浅米灰 / 柔褐
- 已初稿：浅杏 / 淡麦色
- 已去AI：暖橙 / 焦糖杏
- 已审稿：浅茶棕 / 柔暖棕
- 可发布 / 已发布：偏橄榄暖绿或低饱和暖绿
- 发布中：柔和砖红 / 枫叶红

要求：
- 保持不同状态可区分
- 不要出现过冷、过科技感的蓝紫主导
- 整体像同一套“书页状态系统”

**目的：** 统一章节区气质，让状态系统不再破坏整体暖调氛围。

### 4. 处理中章节更像“被摊开的稿纸”

当前处理中主要靠背景变化提示。本轮会增强处理中的视觉状态：
- 处理中卡片底色略亮一点
- 卡片边缘或左侧加入更明确但克制的暖橙强调
- 可加入更柔和的暖色投影
- 不做夸张动画，只做“当前正在桌面上被处理”的感觉

其他章节：
- 不可点击时仍然显示“等待”
- 但视觉上更弱化，不抢当前处理章的注意力

**目的：** 强化当前焦点，增强“写作进行中”的沉浸感。

## 不做的内容

本轮明确不做：
- 不做仪表盘化改造
- 不改页面顶部结构
- 不改侧边栏
- 不新增功能组件
- 不改章节处理接口
- 不改日志流逻辑
- 不调整全站主题变量系统
- 不把明暖风格扩散到 prompt / accounts 页面

## 受影响文件

### 主要修改文件
- `fanqie-workbench/src/web/pages/books-page.tsx`

### 可能复用但不需要改的文件
- `fanqie-workbench/src/web/components/ui/card.tsx`
- `fanqie-workbench/src/web/components/ui/badge.tsx`
- `fanqie-workbench/src/web/styles/tokens.ts`

原则上，本轮应尽量把改动控制在 `books-page.tsx` 内部，通过局部 style 调整完成。

## 具体落地方式

### 章节容器
- 维持现有章节列表结构
- 让每一章从“横向一行”变成更明显的卡片块表现
- 允许保留现有数据结构和映射逻辑

### 章号标签
- 把左侧数字从 `12` 调整为 `第12章`
- 使用更圆润的标签底色与内边距

### 标题层级
- 标题字号略高于当前
- 颜色更偏深棕/暖墨色，而不是纯白
- 辅助文字用浅棕灰，像纸上的淡色注释

### 状态标签
- 保留现有阶段语义映射表位置
- 只改颜色映射与圆角/背景形态

### Hover 态
- Hover 时不做强科技感高亮
- 改为更柔和的暖纸浮起感（轻阴影/边框变化）

### 处理中状态
- 当前处理章卡片明显亮于其他卡片
- “处理中...”文字与卡片强调色统一

## 验证标准

### 功能不回退
- 章节筛选仍可用
- 处理按钮行为不变
- 正在处理中时其他章节仍然显示等待
- 已完成章节仍然显示 ✓
- `LiveLogPanel` 显示逻辑不变

### 视觉结果达成
- 章节区明显脱离“后台表格感”
- 整体更像写作软件 / 章节稿件管理区
- 暖色调统一且克制
- 处理中的章节是视觉焦点
- 不影响暗/浅主题的可读性

## 测试方式

1. 运行：`cd fanqie-workbench && npx tsc --noEmit`
2. 运行：`cd fanqie-workbench && npm test`
3. 打开书籍页，重点检查：
   - 展开某本书后章节区视觉是否明显更柔和
   - 章号标签是否更像章节便签
   - 各阶段 badge 是否统一成暖调体系
   - 点击处理后，当前处理章是否像“被摊开的稿纸”
   - 暗色/浅色主题下都不刺眼、不发灰、不失去层次
